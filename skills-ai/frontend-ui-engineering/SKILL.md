# Frontend UI Engineering

## Overview

Build production-quality user interfaces that are accessible, performant, and visually polished. The goal is UI that looks like it was built by a design-aware engineer — not like it was generated by an AI. This means real design system adherence, proper accessibility, thoughtful interaction patterns, and no generic "AI aesthetic."

## When to Use

- Building new UI components or pages
- Modifying existing user-facing interfaces
- Implementing responsive layouts
- Adding interactivity or state management
- Fixing visual or UX issues

## Petfhans Context

## Component Architecture

### Separation of concerns

```tsx
// Container: handles data
export function PetListContainer({ clinicId }: { clinicId: string }) {
  const [pets, setPets] = useState<Pet[]>([])
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState<string | null>(null)

  useEffect(() => {
    const controller = new AbortController()
    fetch(`/api/vet/pets?clinic_id=${clinicId}`, { signal: controller.signal })
      .then(r => r.ok ? r.json() : Promise.reject(r.statusText))
      .then((data: { pets: Pet[] }) => setPets(data.pets))
      .catch((err: unknown) => {
        if (err instanceof DOMException && err.name === 'AbortError') return
        setError(err instanceof Error ? err.message : 'Error al cargar')
      })
      .finally(() => setLoading(false))
    return () => controller.abort()
  }, [clinicId])

  if (loading) return <PetListSkeleton />
  if (error)   return <ErrorState message={error} />
  if (!pets.length) return <PetListEmpty />
  return <PetList pets={pets} />
}

// Presentation: handles rendering only
export function PetList({ pets }: { pets: Pet[] }) {
  return (
    <ul role="list">
      {pets.map(pet => <PetCard key={pet.id} pet={pet} />)}
    </ul>
  )
}
```

### State management — choose the simplest approach

```
Local state (useState)         → Component-specific UI state
Lifted state                   → Shared between 2-3 siblings
URL state (searchParams)       → Filters, pagination, shareable state
Server state (SWR, fetcher)    → Remote data — prefer Server Components
Context                        → Theme, auth, locale (rare in this app)
```

**Avoid prop drilling deeper than 3 levels.** Restructure or introduce context.

## Avoid the AI Aesthetic

| AI Default | Why It's a Problem | What to Do Instead |
|---|---|---|
| Purple/indigo everything | Clashes with Petfhans coral identity | Use `var(--pf-coral)`, reserve purple for AI only |
| Excessive gradients | The system has ONE gradient (owner hero) | Flat backgrounds with `var(--pf-*)` |
| `rounded-2xl` everywhere | Ignores the radius hierarchy | Use `var(--pf-r-sm/md/lg/xl)` by context |
| Lorem ipsum copy | Hides layout problems | Use realistic Spanish veterinary content |
| Equal padding everywhere | Destroys visual hierarchy | Follow the 4px spacing rhythm |
| Generic card grids | Ignores information priority | Use stat cards, list rows, quick actions — the established patterns |
| Heavy shadows | Competes with content | Max: `var(--pf-shadow-card-hover)` on hover only |
| Emoji as icons | Inconsistent, not accessible | Lucide icons with `size={16}` in nav, `size={18-20}` in chips |

## Loading, Error, and Empty States

Every data-dependent component must handle all three:

```tsx
// Skeleton — pulse animation, not a spinner
function PetListSkeleton() {
  return (
    <div aria-busy="true" aria-label="Cargando mascotas" style={{ display: 'flex', flexDirection: 'column', gap: 8 }}>
      {[1, 2, 3].map(i => (
        <div key={i} style={{
          height: 72, borderRadius: 'var(--pf-r-md)',
          background: 'linear-gradient(90deg, var(--pf-surface) 25%, var(--pf-border) 50%, var(--pf-surface) 75%)',
          backgroundSize: '200% 100%',
          animation: 'shimmer 1.5s infinite',
        }} />
      ))}
    </div>
  )
}

// Error state
function ErrorState({ message }: { message: string }) {
  return (
    <div role="alert" style={{ padding: '40px 20px', textAlign: 'center' }}>
      <p style={{ font: 'var(--pf-text-body)', color: 'var(--pf-danger-fg)' }}>{message}</p>
    </div>
  )
}

// Empty state — always with CTA
function PetListEmpty() {
  return (
    <div style={{ padding: '48px 20px', textAlign: 'center' }}>
      <div style={{
        width: 56, height: 56, borderRadius: 'var(--pf-r-lg)',
        background: 'var(--pf-coral-soft)', color: 'var(--pf-coral)',
        display: 'flex', alignItems: 'center', justifyContent: 'center',
        margin: '0 auto 16px',
      }}>
        <PawPrint size={28} strokeWidth={1.5} />
      </div>
      <p style={{ font: 'var(--pf-text-h3)', color: 'var(--pf-ink)', margin: '0 0 6px' }}>
        Sin mascotas registradas
      </p>
      <p style={{ font: 'var(--pf-text-body)', color: 'var(--pf-muted)', margin: '0 0 20px' }}>
        Registra la primera mascota de la clínica.
      </p>
      <Link href="/vet/pets/new" className="pf-btn pf-btn-primary">
        + Nueva mascota
      </Link>
    </div>
  )
}
```

## Responsive Design

| Portal | Approach | Key breakpoint |
|---|---|---|
| `/vet/*` | Desktop-first, sidebar 220px fixed | `@media (max-width: 767px)` — hamburger menu |
| `/admin/*` | Desktop-first, sidebar 220px fixed | `@media (max-width: 767px)` — hamburger menu |
| `/owner/*` | **Mobile-first** | `@media (min-width: 768px)` — 2-column layout |

Test at: 390px (iPhone 14), 768px (iPad), 1280px (laptop), 1440px (desktop).

## Optimistic Updates

```tsx
// For actions where the user expects immediate feedback
const [localActive, setLocalActive] = useState(pet.is_active)

async function handleToggle() {
  setLocalActive(prev => !prev) // Optimistic update
  try {
    await fetch(`/api/vet/pets/${pet.id}`, {
      method: 'PATCH',
      body: JSON.stringify({ is_active: !localActive }),
      headers: { 'Content-Type': 'application/json' },
    })
  } catch {
    setLocalActive(prev => !prev) // Revert on error
  }
}
```

## Verification Checklist

```
□ Component renders without console errors
□ All interactive elements keyboard-accessible (Tab through the page)
□ Loading, error, and empty states all handled
□ Follows design system: var(--pf-*) tokens, Lucide icons, correct radii
□ No hardcoded hex colors or pixel values that should be tokens
□ No emoji as UI icons
□ Responsive: works at 390px and 1280px
□ No accessibility warnings in axe DevTools extension
□ npx tsc --noEmit passes
□ npx eslint src/path/to/file passes with 0 errors
```
